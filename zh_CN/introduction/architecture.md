# 系统架构

Datalayers 采用存算分离的分布式架构，将计算层与存储层解耦，实现资源独立扩展与弹性伸缩。

## 架构概览

Datalayers 的整体架构自上而下分为以下层次：

### 接入层

接入层提供多种协议支持，便于不同客户端和工具接入：

- **SQL 协议**：兼容 MySQL 方言，支持标准 SQL 查询
- **Arrow Flight SQL**：高性能列式数据传输协议
- **PostgreSQL 协议**：兼容 PostgreSQL 网络协议
- **HTTP/REST API**：提供 RESTful 接口
- **InfluxDB 行协议**：兼容 InfluxDB 写入协议
- **Redis 协议**：兼容 Redis 命令协议，用于键值存储操作
- **Prometheus Remote Write/Read**：兼容 Prometheus 生态

### 计算层

计算层负责 SQL 解析、查询优化和执行：

- **查询解析与优化器**：将 SQL 解析为执行计划，并进行基于规则和代价的优化
- **执行引擎**：基于向量化执行模型，高效完成数据扫描、聚合、排序等操作
- **缓存机制**：内置多级缓存（索引缓存、混合缓存、Last 缓存），加速热点数据查询
- **弹性扩展**：计算节点支持水平扩展，可按需增减节点数量

### 存储层

存储层提供多模态数据的统一存储能力：

- **时序存储引擎**：针对时序数据优化，支持高吞吐写入与高压缩比存储
- **日志存储引擎**：支持全文索引，实现海量日志高效检索
- **键值存储引擎**：提供分布式持久化 Key-Value 存储
- **向量存储引擎**：支持高维向量索引与相似度检索
- **对象存储集成**：支持将数据存储至 S3 兼容的对象存储服务（如阿里云 OSS、AWS S3、MinIO 等）

### 协调层

在集群模式下，协调层负责：

- **元数据管理**：维护表结构、分区信息等集群元数据
- **数据分片与路由**：将数据按分区规则分布到不同存储节点
- **节点调度**：协调计算节点与存储节点的资源分配和故障恢复

## 核心设计理念

- **存算分离**：存储与计算资源独立扩展，降低成本，提升资源利用率
- **多模融合**：在统一架构下支持时序、键值、日志、向量等多种数据模型
- **协议兼容**：通过多协议支持降低用户接入和迁移成本
- **弹性伸缩**：各层组件均支持水平扩展，灵活应对业务负载变化
